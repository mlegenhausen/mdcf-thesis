\chapter{Zusammenfassung und Ausblick}
\label{cha:zusammenfassung}
Im Rahmen dieser Masterarbeit wurde ein dreiteiliges Software-System
entwickelt, dass das transparente und anonyme Sammeln von Sensordaten auf
Android-Smartphones ermöglicht. Diese drei Teile sind der {\em Mobile Data
Collector}, das {\em Mobile Data Collection Framework} sowie ein
serverseitiges Framework zum Empfang der gesammelten Sensordaten.

Kern der Arbeit war es eine vertrauenswürdige Plattform zu schaffen, für die
Entwicklung von Plug-Ins zum Sammeln von Sensordaten (potenziell
personenbezogenen), die es aber dem Benutzer erlaubt, alle Aktivitäten und
gesammelten Daten des Plug-Ins einzusehen. Durch diese Transparenz sollen
Smartphone-Besitzer, Vertrauen in den {\em Mobile Data Collector} erlangen und
einen Teil ihrer persönlichen Daten an Plug-Ins preisgeben, ohne einen direkten
Mehrwert von den Plug-Ins zu erhalten. Als Beispiel sei hier Google Maps zu
erwähnen, bei dem man seinen Standort auf einer Karte angezeigt
bekommen, wenn man seine aktuelle Position freigibt.

Die Realisierung der erarbeiteten Anforderungen und Konzepte hat eine
erweiterbare Android-Anwendung hervorgebracht, die das bereits existierende
Security-System von Android durch Überwachungsmechanismen erweitert. Der {\em
Mobile Data Collector} lässt sich durch Sensordaten sammelnde Plug-Ins
erweitern, die mit dem {\em Mobile Data Collection Framework} entwickelt werden.
Vor der ersten Ausführung eines Plug-In werden diese analysiert und der Benutzer
wird über Gefahren, die vom Plug-In ausgehen können informiert. Der Benutzer
behält hierbei, wie im Konzept beschrieben, immer die Kontrolle über seine
Daten. Er kann die Ausführungszeit eines Plug-In bestimmen und wird über die
Ausführung informiert, kann einsehen, auf welche personenbezogenen Daten
zugegriffen wurden sowie welche Daten an den Plug-In spezifischen Server
übertragen werden sollen. Die Anonymität des Benutzers wird dadurch
sichergestellt, dass das Framework den Zugriff auf eindeutige Benutzerdaten gar
nicht erst ermöglicht. Die bei der Übertragung verwendete eindeutige
Benutzerkennung erlaubt zwar die Zuordnung von Daten zu bereits empfangen
Daten, erlaubt aber nicht die Zuordnung zu realen Personen.

Die Evaluation hat gezeigt, dass sich mit dem {\em Mobile Data Collection
Framework} Plug-Ins für reale Anwendungsszenarien entwickeln lassen. So ist die
Aufnahme von Positionsdaten, die wohl wichtigste Art von Daten im Bereich der
mobilen Datenerfassung, relativ einfach möglich. Auch lässt sich das Plug-In,
einfach um die Erfassung zusätzlicher Daten erweitern. Hierbei wurde bewusst
darauf geachtet, den Entwickler so wenig wie möglich bei der Entwicklung von
Plug-Ins einzuschränken. Die Evaluation zeigt aber auch die zurzeit bestehenden
Überwachungsgrenzen des {\em Mobile Data Collector}. So ist eine Überwachung der
Kamera und des Mikrofons nicht bzw. nur mit relativ großem Aufwand möglich.
Dieses Problem lässt sich am besten durch die Einführung von OSGi lösen. Hierbei
bleibt aber abzuwarten bis Frameworks wie Dynamix den nötigen Reifegrad erreicht
haben, um diese im produktiven Einsatz zu verwenden.

Das zukünftige Anwendungsgebiet des {\em Mobile Data Collector} liegen in
erster Linie im Bereich der Forschung. Hier existiert eine Vielzahl von
Projekten, die auf die Erhebung von personenbezogenen Daten angewiesen sind.
Aber auch in der Industrie ergeben sich viele Anwendungsfelder z.~B. im Bereich
des Marketings.

Abschließend bleibt zu hoffen, dass diese Arbeit generell einen Denkanreiz
liefern kann, Software zu entwickeln, die Transparenz anbieten und die Macht
über seine personenbezogenen Daten dem Benutzer zurückgeben. Es wurde
gezeigt, dass es hierbei nur bedingt technische Probleme gibt.
