% Kurzfassung und Abstract

\centerline{\bf Kurzfassung}
\bigskip
In der heutigen Zeit erfassen Smartphones sehr viele personenbezogene Daten
(z.~B. Standort, Benutzerverhalten) über ihren Besitzer. Diese Daten besitzen
einen hohen Wert für öffentliche Einrichtungen und die Industrie. In vielen
Fällen werden diese Daten ohne das Wissen des Benutzers von Herstellern von
Hardware, Betriebssystem und Applikationen gesammelt und an diese übertragen. 
Im Rahmen dieser Arbeit wurde ein Framework für das Smartphone-Betriebssystem
Android entwickelt welches die Erstellung von Anwendungen zum transparenten und
anonymen Sammeln personenbezogener Daten ermöglicht. Der Benutzer erhält die
Möglichkeit zu jedem Zeitpunkt alle gesammelten Daten einzusehen und kann selbst
bestimmen, welche Daten zur Übertragung freigegeben werden sollen und welche
nicht. Die Verwendung des Frameworks stellt sicher, dass der Benutzer die
Kontrolle über seine persönlichen Daten hat. Anwendungen auf Basis des
entwickelten Frameworks können somit von einem gestärkten Vertrauen der Benutzer
profitieren.

\centerline{\bf Abstract}
\bigskip
Today's smartphones collect a high amount of personalized data (e.g. position,
user behaviour) from the owner. This data has a high value for public
institutions and the industry. But in most cases this data is collected and
transmitted without the knowledge of the user to manufacturers of hardware,
operating systems and applications. This paper focusses the development of a
framework for the smartphone operating system Android which enables the creation
of applications that collect personalized data in a transparent and anonymous
way. At any point in time the user has the possibility to review all
the collected data and he can decide which data he wants to approve for
transfer and which not. The usage of the framework guaranteed that the user is in
control of his personalized data.

\section*{Aufgabenstellung}
Die \acf{boinc} ermöglicht es die ungenutzte Rechenleistung vieler tausender
Computer für die Lösung komplexer wissenschaftlicher Probleme zu nutzen. Dabei
installieren Benutzer auf Ihren privat oder beruflich genutzten Computern die
\acs{boinc} Software und in Zeiten, in denen die Rechner nicht ausgelastet sind,
nutzt \acs{boinc} die Rechenzeit zur Lösung wissenschaftlicher Probleme. Welchem
wissenschaftlichen Projekt man seine Rechenleistung zur Verfügung stellt bleibt
dem Benutzer selbst überlassen.

In Anlehnung an dieses Prinzip soll im Rahmen dieser Arbeit eine Anwendung
geschaffen werden welche es ermöglicht die auf Smartphones anfallenden
(potenziell personenbezogenen) Daten wissenschaftlichen Projekten zur Verfügung
zu stellen. Dies können zum Beispiel die aktuelle Temperatur, der Lärmpegel der
Umgebung, die aktuelle Position, die verfügbaren Netzwerkverbindungen oder die
Daten des Beschleunigungssensors sein. Die Erfassung und Übertragung von Daten
soll dabei möglichst transparent für den Benutzer sein, sodass dieser Kenntnis
und volle Kontrolle über die erhobenen Daten hat. Die Lösung soll dabei die
Realisierung einer Vielzahl unterschiedlicher Datenerfassungsanwendungen
ermöglichen. Das Smartphone-Betriebssystem Android soll technische Basis der zu
entwickelnden Lösung sein.

Im Rahmen einer Anforderungsanalyse sollen zunächst Anforderungen an Transparenz
und Anonymität sowie technische Anforderungen abgeleitet werden. Anschließend
sollen in der Konzeptionsphase mögliche Technologien zur Umsetzung evaluiert und
eine dieser für die Realisierung ausgewählt werden. Auf Basis der gewählten
Technologie erfolgt dann ein Programmentwurf welcher in der
Implementierungsphase in ein ausführbares Programm überführt werden soll. Anhand
zweier zu implementierender Demonstrator-Anwendungen soll abschließend im Rahmen
der Evaluation gezeigt werden dass die entwickelte Lösung für reale
Anwendungsszenarien geeignet ist.
